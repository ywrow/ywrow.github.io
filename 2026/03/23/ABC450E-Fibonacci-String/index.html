ABC450E Fibonacci String

思路

这道题要处理长度为 101810^{18}1018 的字符串，直接构造肯定不行。但仔细观察生成规则 Si=Si−1+Si−2S_i = S_{i-1} + S_{i-2}Si​=Si−1​+Si−2​，发现字符串长度的增长符合斐波那契数列。

1. 长度极长

斐波那契数列增长是指数级的。即使 ∣X∣,∣Y∣|X|, |Y|∣X∣,∣Y∣ 只有 1，大概不到 90 项，长度就会超过 101810^{18}1018。
这意味着对于任何 k≥90k \ge 90k≥90，字符串 SkS_kSk​ 的前 101810^{18}1018 个字符的结构是完全一样的。
题目问的是 S1018S_{10^{18}}S1018​，其实我们只需要算到第一个长度超过 101810^{18}1018 的那一层（记为 max_k），后面的层数在查询范围内等价于这一层。

2. 预处理信息

我们可以先递推预处理出前 90 层左右的信息：

  • len[k]：第 kkk 层的长度。注意如果超过 101810^{18}1018 就直接存成一个很大的数（比如 2×10182 \times 10^{18}2×1018），防止 long long 溢出。
  • cnt[k][c]：第 kkk 层中字符 c 出现的总次数。递推式很简单：cnt[k][c] = cnt[k-1][c] + cnt[k-2][c]。

3. 递归查询

要求 [L,R][L, R][L,R] 区间内字符 CCC 的个数，可以转化为前缀和相减：f(k, r, c) - f(k, l-1, c)。
其中 f(k, l, c) 表示在第 kkk 层字符串的前 limit 个字符中，c 出现了多少次。

利用 Sk=Sk−1+Sk−2S_k = S_{k-1} + S_{k-2}Sk​=Sk−1​+Sk−2​ 的结构递归：

  1. 边界：如果 k=1k=1k=1 或 k=2k=2k=2，直接遍历原始字符串 XXX 或 YYY 统计前 l 个。
  2. 在左半边：如果 l ≤\le≤ len[k-1]，说明目标全在 Sk−1S_{k-1}Sk−1​ 里，递归 f(k-1, l, c)。
  3. 跨越两边：如果 l > len[k-1]，说明包含了完整的 Sk−1S_{k-1}Sk−1​ 和 Sk−2S_{k-2}Sk−2​ 的一部分。
    答案 = (Sk−1S_{k-1}Sk−1​ 中 c 的总数) + f(k-2, limit - len[k-1], c)。
